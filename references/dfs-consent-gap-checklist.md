## Governing Principle

> **Pilot status is not automatic consent.** Being a contracted pilot customer does not give AiQ permission to model DFS's data, generate a reward-AI strategy attributed to DFS, produce a board report in DFS's name, or store and share those outputs. Each of these activities requires explicit written confirmation in the pilot agreement.

This checklist must be reviewed against the current DFS pilot agreement before any outputs are circulated. Any gap must be closed with written sign-off from DFS before outputs leave the building.

---

## Scope of This Checklist

The DFS evidence pack includes the following outputs that require consent confirmation:

| Output | Description |
|---|---|
| Company profile | DFS headcount, revenue, payroll, sector, and HR structure modelled in the AiQ platform |
| Reward-AI strategy | AI-generated strategy attributed to DFS, including vision, principles, initiative portfolio, and strategic shifts |
| Board report | AI-generated board report in DFS's name, intended for board/RemCo/leadership audience |
| Business case | Financial model (cost, value, ROI) attributed to DFS |
| Capability assessment | DFS reward team's capability gaps and enablement actions |
| Evidence pack | All of the above compiled into a shareable document |

## Instructions for Completing This Checklist

1. Obtain the current DFS pilot agreement from the commercial/legal team.
2. Review each item in sections A–D against the agreement text.
3. For each item, mark the status as one of:
   - **Confirmed** — the agreement explicitly covers this item
   - **Gap** — the agreement does not cover this item or is ambiguous
   - **Not Applicable** — this item does not apply to the current scope
4. For each Gap, add a row to the Gap Assessment table and assign an owner and target date.
5. For each Gap, obtain written sign-off from DFS before the relevant output is circulated.
6. Complete the sign-off section when all gaps are closed.
7. Link this document in the evidence pack index.

## Why This Is a P0 Gate

The brief states: *"P0 items gate release of any further DFS evidence."*

Circulating a board report, strategy, or business case attributed to DFS without confirmed consent creates legal, reputational, and commercial risk for both AiQ and DFS. The figures in the report (headcount, revenue, payroll, strategic direction) are commercially sensitive. The board report is intended for DFS's board or RemCo — an audience that will act on its contents.

No DFS-attributed output should leave the building until this checklist is complete and all gaps are closed.
